# 策略2说明

## 策略逻辑

策略2是一个基于前一个交易日表现和当日盘中表现的选股策略，具体条件如下：

### 1. 前一个交易日条件
- **涨幅要求**：前一个交易日股价涨幅在2%-8%之间
- **成交量要求**：前一个交易日成交量是5日成交量均值(vol5)的1.2倍以上

### 2. 当日盘中条件
- **涨幅要求**：当日盘中涨幅在-2%~2%之间（即小幅波动）

## 策略特点

1. **数据完整性要求**：策略运行前提是前一个交易日数据完整（前一个交易日收盘后已经获取新数据）
2. **盘中运行**：策略在当日盘中运行，使用实时行情数据
3. **双重筛选**：结合前一日表现和当日盘中表现进行筛选
4. **风险控制**：选择前一日表现良好但当日波动较小的股票

## 使用流程

### 数据准备
1. 收盘后运行 `main.py` 更新日线数据和技术指标
2. 确保数据库中有前一个交易日的完整数据

### 策略执行
1. 盘中运行 `run_strategy2.py` 执行策略2
2. 策略会自动从多个数据源获取实时行情数据

### 输出结果
策略2的输出包括：
- 股票代码和名称
- 今日盘中涨幅和现价
- 前一日收盘价和涨幅
- 前一日成交量和5日成交量均值
- 前一日量比（成交量/vol5）
- 股票所属行业

## 文件说明

- `strategy.py` - 包含策略2的实现方法 `screen_stocks_strategy2()` 和 `print_results_strategy2()`
- `run_strategy2.py` - 策略2的执行入口
- `selected_stocks_strategy2.txt` - 策略2的选股结果保存文件

## 策略逻辑详解

### 数据获取
1. 从数据库获取前一个交易日的日线数据和技术指标
2. 从新浪财经/腾讯财经/Tushare获取当日实时行情数据
3. 将前一日数据与当日实时数据进行匹配

### 筛选条件
1. **前一日涨幅筛选**：2% ≤ 前一日涨幅 ≤ 8%
2. **前一日量比筛选**：前一日成交量 ≥ vol5 × 1.2
3. **当日涨幅筛选**：-2% ≤ 当日涨幅 ≤ 2%
4. **数据完整性检查**：确保所有必要数据都存在且有效

### 结果排序
选出的股票按当日涨幅从高到低排序

## 注意事项

1. **数据更新时间**：确保前一日数据已更新，建议在收盘后运行数据更新
2. **实时数据源**：策略会尝试多个数据源，确保能获取到实时行情
3. **盘中时间**：策略设计为盘中运行，建议在交易时间内执行
4. **数据验证**：策略会检查数据的完整性和有效性，跳过无效数据
